# 独立服务器实现方案分析

## 一、项目概述

### 1.1 目标

将Android客户端的音频收发功能迁移到独立服务器，实现以下目标：

- **24小时在线**：服务器持续运行，无需手机在线
- **实时收听**：接收频道内的音频数据
- **发送音频**：将音频数据发送到频道
- **数据存储**：保存接收到的音频数据供后续回放分析

### 1.2 现有架构分析

```
┌─────────────┐      gRPC/HTTP       ┌──────────────────┐
│  Android    │ ──────────────────►  │   RPC服务器      │

└─────────────┘                      └──────────────────┘
       │                                      │
       │ 语音TCP连接                           │
       ▼                                      ▼
┌─────────────┐                       ┌──────────────────┐
│  对讲机/    │                       │   语音服务器     │
```

## 二、核心技术分析

### 2.1 音频数据类型

根据代码分析，系统支持以下音频类型：

| 类型 | 描述 | 用途 |
|------|------|------|
| `TX_AUDIO` | 发射音频 | 用户说话时的音频数据 |
| `TX_AUDIO_STOP` | 发射停止 | 停止发射信号 |
| `RX_AUDIO` | 接收音频 | 接收到的其他用户音频 |
| `RX_AUDIO_STOP` | 接收停止 | 停止接收信号 |

**文件位置**：`app/src/main/java/p298t3/EnumC6128n1.java`

### 2.2 音频编解码

#### 2.2.1 Opus编解码

- **OpusEncoder**：`com.p040dw.audio.codec.OpusEncoder`
- **OpusDecoder**：`com.p040dw.audio.codec.OpusDecoder`
- 采样率：48000Hz
- 应用场景：实时语音通信

#### 2.2.2 SBC编解码（蓝牙）

- **SbcEncoder**：`com.p040dw.sbc.SbcEncoder`
- **SbcDecoder**：`com.p040dw.sbc.SbcDecoder`
- 比特率：32000bps
- 应用场景：蓝牙数据传输

#### 2.2.3 AFSK解调

- **Decoder**：`com.p040dw.mdc.Decoder`
- **MDecoder**：`com.p040dw.multimon.MDecoder`
- 应用场景：无线电AFSK信号解调

#### 2.2.4 SSTV编解码

- **Robot36**：`xdsopl.robot36` - SSTV图像解码
- **SSTV Encoder**：`om.sstvencoder` - SSTV图像编码
- 支持模式：Martin1, Martin2, Scottie1, Scottie2, Robot36等

**文件位置**：
- `app/src/main/java/com/p040dw/audio/codec/`
- `app/src/main/java/com/p040dw/sbc/`
- `app/src/main/java/com/p040dw/mdc/`

### 2.3 语音服务器连接

#### 2.3.1 获取连接参数

通过gRPC调用获取语音服务器地址：

```java
// 获取频道连接参数
GetChannelConnectionParmRequest request = GetChannelConnectionParmRequest.newBuilder()
    .setChannelID(channelId)
    .build();
GetChannelConnectionParmResult result = ihtStub.getChannelConnectionParm(request);
```

**返回参数**：
- `ip`: 语音服务器IP
- `port`: 语音服务器端口
- `bitRates`: 语音比特率
- `auth`: 用户在频道中的权限
- `rfCh`: 无线电配置（频率、亚音等）

**proto定义**：
```protobuf
message GetChannelConnectionParmRequest {
  uint64 channelID = 1;
}

message GetChannelConnectionParmResult {
  string ip = 2;
  int32 port = 3;
  int32 bitRates = 4;
  ChannelMemberAuth auth = 5;
  RfChannelFields rfCh = 6;
}
```

### 2.4 数据传输协议

#### 2.4.1 TCP二进制协议

音频数据通过TCP Socket传输，使用二进制格式：

```
┌─────────────────────────────────────────────────┐
│  音频数据类型 (1字节)  │  数据长度 (4字节)       │
├─────────────────────────────────────────────────┤
│               音频数据 (可变长度)                │
└─────────────────────────────────────────────────┘
```

数据类型枚举：
- `TX_AUDIO = 1` - 发送音频
- `TX_AUDIO_STOP = 2` - 发送停止
- `RX_AUDIO = 3` - 接收音频
- `RX_AUDIO_STOP = 4` - 接收停止
- `SET_SIGN_DATA = 5` - 设置信号数据

#### 3.2.2 TCP语音服务器模块

**功能**：
- 维护与客户端的TCP长连接
- 音频数据转发
- 多用户语音混音

**核心逻辑**：

```
TCP语音服务器工作流程：

1. 监听端口（从GetChannelConnectionParm获取）
2. 客户端连接认证
3. 加入频道
4. 循环处理：
   - 接收客户端音频数据
   - 音频处理（解码→重采样→编码）
   - 转发给频道内其他用户
   - 存储音频数据
5. 客户端断开
6. 退出频道
```

**音频处理流程**：


#### 3.2.4 音频处理模块

**核心类设计**：

```python
class AudioProcessor:
    """音频处理器"""
    
    def __init__(self):
        # Opus编解码器
        self.encoder = OpusEncoder(sample_rate=48000, bitrate=32000)
        self.decoder = OpusDecoder(sample_rate=48000)
        
        # 重采样器
        self.resample_8k_to_32k = Resample(8000, 32000)
        self.resample_32k_to_8k = Resample(32000, 8000)
    
    def encode(self, pcm_data: bytes) -> bytes:
        """PCM数据编码为Opus"""
        return self.encoder.encode(pcm_data)
    
    def decode(self, opus_data: bytes) -> bytes:
        """Opus解码为PCM"""
        return self.decoder.decode(opus_data)
    
    def resample_8k_to_32k(self, pcm_data: bytes) -> bytes:
        """8kHz采样率转换为32kHz"""
        return self.resample_8k_to_32k.process(pcm_data)
    
    def resample_32k_to_8k(self, pcm_data: bytes) -> bytes:
        """32kHz采样率转换为8kHz"""
        return self.resample_32k_to_8k.process(pcm_data)
```
