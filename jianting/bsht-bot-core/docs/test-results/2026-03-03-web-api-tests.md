## 已知问题与修复建议

### 1. Toggle API 请求体格式问题
**问题:** toggle相关API使用FastAPI的自动参数解析，但测试中使用了错误的请求格式

**当前实现:**
```python
@router.post("/rules/toggle")
async def toggle_rule(rule_id: int, db: Database = Depends(get_db)):
```

**建议修复:**
```python
from fastapi import Body

@router.post("/rules/toggle")
async def toggle_rule(
    rule_id: int = Body(..., embed=True),
    db: Database = Depends(get_db)
):
```

或者修改测试使用正确的请求格式：
```python
# 测试中应该使用
response = client.post("/api/rules/toggle", json={"rule_id": 1})
```

### 2. 批量删除API空列表处理
**问题:** 批量删除空列表时返回422而非400

**建议:** 在API中添加空列表验证并返回400错误

### 3. 404错误处理不一致
**问题:** 某些API在资源不存在时返回200而非404

**建议:** 统一错误处理，确保资源不存在时返回404

### 4. 推送服务详情缺少user_count字段
**问题:** API响应中没有返回user_count字段

**建议:** 在服务详情API中添加用户数统计
