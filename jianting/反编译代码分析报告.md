## 七、关键调用关系

### 7.1 登录认证流程
```
用户点击登录 → LoginActivity.onClick() 
    → 调用对应SDK（QQ/Google/微信）
    → 获取OpenID和AccessToken
    → 调用UserGrpc.OpenIDLogin()
    → 服务器返回LoginResult（包含双令牌）
    → 令牌存储到C5983b/C2252b
    → 跳转到主界面
```

### 7.2 消息接收流程
```
gRPC长连接 → IMGrpc.IMStub.receiveMessage() 
    → C2836f.mo9054c()（接收回调）
    → C2208a.m9922b()（消息分发）
    → 根据消息类型处理：
        - 用户状态更新 → 更新本地用户状态
        - 文本消息 → 显示到UI
        - AFSK数据 → 解码并播放/处理
```

### 7.3 消息发送流程
```
用户输入/触发发送 → C3162c.m16821a()构建消息
    → C2836f.m15559p()发送消息
    → IMGrpc.IMStub.sendMessage() gRPC调用
    → 服务器处理并返回SendMessageResult
    → C2836f.mo9054c()发送结果回调
```

## 八、安全分析

### 8.1 认证安全
- **双令牌机制：** Access Token + Refresh Token
- **令牌过期检查：** 自动刷新机制
- **多因素认证：** 支持密码、第三方OAuth

### 8.2 通信安全
- **gRPC TLS：** 默认使用加密通道
- **端到端加密：** 可能用于敏感数据（需进一步验证）
- **心跳机制：** 连接状态监控

### 8.3 潜在风险
1. **令牌存储安全：** 需要验证是否使用Android Keystore
2. **证书固定：** 需要验证gRPC是否实现证书固定
3. **日志泄露：** 日志中可能包含敏感信息
